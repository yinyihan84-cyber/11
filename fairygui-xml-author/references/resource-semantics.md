# 资源语义

## 用途

记录切图和资源如何进入 FairyGUI XML。资源不是只有文件名和尺寸，还可能带有九宫格、平铺、平滑、白模染色和运行时替换语义。

## image 与 loader 的选择

- 固定静态切图用 `image`。
- 头像、奖励、皮肤、广告图、动态背景、运行时图标用 `loader`。
- 纯切图模式且用户要求“不要把图片直接写进 XML”时，统一使用 `loader` 以 `url="ui://..."` 引用资源。
- 缺图但需要先保留结构时，使用空 `loader` 占位，不要用 `graph` 脑补。

## 九宫格判断

显示尺寸与原图尺寸不一致时，检查视觉角色：

- 面板底、按钮底、卡片底、标签底、进度槽、边框、端帽、圆角、描边、阴影、内凹区域，通常需要九宫格。
- 纯图标、插画、照片、文字图、徽章主体和不可拆分装饰，通常不补九宫格。
- 非等比缩放且无法确认语义时，标记“拉伸语义待确认”。

写法：

```xml
<image id="progress_bg" name="progress_bg.png" path="/res/" scale="9grid" scale9grid="11,12,5,4"/>
```

## 平铺判断

连续纹理、条纹、格子底、噪声底等需要重复铺满时，平铺语义写在 `package.xml` 图片资源声明上。

写法：

```xml
<image id="tile_bg" name="tile_bg.png" path="/res/" scale="tile"/>
```

`gridTile` 不是平铺主标记，只在样本或资源面板明确给出 tile grid 索引时补写。

## 白模染色

白模或灰模资源需要在显示节点上染色：

```xml
<loader id="n1_bar" name="bar" size="271,24" url="ui://pkgidprogress_fill" fill="scaleFree" color="#fcd00b"/>
```

不要因为资源本身是白色就输出白色；以参考图或样本 XML 的最终视觉色为准。

## 纯切图约束

当用户要求参考效果图拼界面且提供切图目录时：

- 不允许自行创建 `graph` 代替背景、按钮、描边、底板、阴影或装饰。
- 不允许用文本、颜色块或程序化图形脑补未提供的美术。
- 可显示视觉元素优先引用已有切图。
- 缺失资源用空 `loader` 占位，并在交付说明中列出缺图。

## 包裹关系

写“底 / 框 / bg / frame + 内容”的组合时，底图或框图区域应大于或等于内部 icon / text 的视觉区域，内部内容内缩居中。不要让 `reward_icon`、`button_text`、`badge_icon` 等内容节点尺寸反而大于对应底框，除非参考图明确是溢出装饰。
